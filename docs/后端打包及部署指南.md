## 🔧 系统要求

### 最低配置

- **CPU**: 2 核心
- **内存**: 2 GB RAM
- **磁盘**: 20 GB 可用空间
- **操作系统**: Linux (Ubuntu 20.04+ / CentOS 7+ / Debian 10+)

### 推荐配置

- **CPU**: 4 核心
- **内存**: 4-8 GB RAM
- **磁盘**: 50 GB+ 可用空间（SSD 推荐）
- **操作系统**: Ubuntu 22.04 LTS

### 软件要求

**开发环境（Windows）：**
- **Go**: 1.21+ （用于交叉编译）
- **Git**: （可选，用于版本控制）

**生产环境（Linux）：**
- **PostgreSQL**: 12+
- **外部工具**: httpx, katana（可选）
- **进程管理**: systemd（推荐）或 supervisor

## 💻 开发环境说明

### Windows 开发环境

本文档假设您在 **Windows** 上进行开发，需要交叉编译生成 **Linux** 二进制文件用于部署。

**优势：**
- 在熟悉的 Windows 环境下开发
- Go 支持交叉编译，无需 Linux 环境
- 编译后的二进制文件可直接部署到 Linux 服务器

**准备工作：**
1. 在 Windows 上安装 Go 1.21+
2. 配置 Go 环境变量
3. 验证安装：`go version`

**验证 Go 安装：**

```powershell
# PowerShell
go version
# 应该显示: go version go1.21.x windows/amd64

# 检查 Go 环境
go env GOPATH
go env GOROOT
```

## 📦 打包步骤

> **开发环境说明**：本文档假设开发环境为 Windows，最终部署到 Linux 服务器。需要在 Windows 上进行交叉编译生成 Linux 二进制文件。

### 方式一：Windows 开发环境打包（推荐）

#### 1. 准备代码

**Windows PowerShell 或 CMD:**

```powershell
# 进入项目目录
cd C:\Users\red\Desktop\WebCheckly.com\backend

# 确保代码是最新的（如果使用 Git）
git pull origin main

# 检查 Go 版本
go version  # 需要 1.21+
```

#### 2. 安装依赖

```powershell
# 下载依赖
go mod download

# 整理依赖（可选）
go mod tidy
```

#### 3. 交叉编译 Linux 二进制文件

**Windows PowerShell:**

```powershell
# 方式1: 基本交叉编译（Linux amd64）
$env:GOOS="linux"
$env:GOARCH="amd64"
go build -o webcheckly main.go

# 方式2: 优化编译（去除调试信息，减小体积，推荐）
$env:GOOS="linux"
$env:GOARCH="amd64"
$env:CGO_ENABLED="0"
go build -ldflags="-s -w" -o webcheckly main.go

# 方式3: 一键编译脚本（推荐）
# 创建 build.ps1 文件
```

**Windows CMD:**

```cmd
REM 设置环境变量并编译
set GOOS=linux
set GOARCH=amd64
set CGO_ENABLED=0
go build -ldflags="-s -w" -o webcheckly main.go
```

**创建编译脚本（推荐）：**

在 `backend` 目录创建 `build-linux.ps1`：

```powershell
# build-linux.ps1 - Windows 下编译 Linux 版本
Write-Host "开始编译 Linux 版本..." -ForegroundColor Green

# 设置编译环境
$env:GOOS = "linux"
$env:GOARCH = "amd64"
$env:CGO_ENABLED = "0"

# 编译
Write-Host "编译参数: GOOS=$env:GOOS, GOARCH=$env:GOARCH, CGO_ENABLED=$env:CGO_ENABLED" -ForegroundColor Yellow
go build -ldflags="-s -w" -o webcheckly main.go

if ($LASTEXITCODE -eq 0) {
    Write-Host "编译成功！" -ForegroundColor Green
    $file = Get-Item webcheckly -ErrorAction SilentlyContinue
    if ($file) {
        Write-Host "文件大小: $([math]::Round($file.Length / 1MB, 2)) MB" -ForegroundColor Cyan
        Write-Host "文件路径: $($file.FullName)" -ForegroundColor Cyan
    }
} else {
    Write-Host "编译失败！" -ForegroundColor Red
    exit 1
}
```

执行编译脚本：

```powershell
# PowerShell
.\build-linux.ps1
```

#### 4. 验证编译结果

```powershell
# 检查文件是否存在
Test-Path webcheckly

# 查看文件信息
Get-Item webcheckly | Select-Object Name, Length, LastWriteTime

# 查看文件大小（通常应在 20-50 MB 左右）
(Get-Item webcheckly).Length / 1MB
```

**注意**：在 Windows 上编译的 Linux 二进制文件无法在 Windows 上直接运行，需要上传到 Linux 服务器后才能执行。

#### 4. 验证编译结果

```bash
# 检查文件
ls -lh webcheckly
file webcheckly  # Linux
./webcheckly --help  # 如果支持

# 查看文件大小（通常应在 20-50 MB 左右）
```

#### 5. 准备部署包

**Windows PowerShell:**

```powershell
# 创建部署目录
New-Item -ItemType Directory -Force -Path "deploy\webcheckly-backend"
cd deploy\webcheckly-backend

# 复制二进制文件（从 backend 目录）
Copy-Item ..\..\backend\webcheckly .

# 复制迁移文件
Copy-Item -Recurse ..\..\backend\migrations .

# 创建必要的目录
New-Item -ItemType Directory -Force -Path "logs"
New-Item -ItemType Directory -Force -Path "config"

# 创建启动脚本（Linux 使用）
@"
#!/bin/bash
cd `$(dirname `$0)
./webcheckly
"@ | Out-File -FilePath "start.sh" -Encoding UTF8 -NoNewline

Write-Host "部署包已准备完成！" -ForegroundColor Green
Write-Host "目录结构:" -ForegroundColor Cyan
Get-ChildItem -Recurse | Select-Object FullName
```

**或者使用压缩包方式：**

```powershell
# 创建部署压缩包
Compress-Archive -Path webcheckly, migrations, start.sh -DestinationPath webcheckly-backend.zip -Force
Write-Host "部署包已创建: webcheckly-backend.zip" -ForegroundColor Green
```

### 方式二：在 Linux 服务器上直接编译（备选）

如果需要在 Linux 服务器上直接编译，可以使用以下步骤：

```bash
# 在 Linux 服务器上
cd /path/to/WebCheckly.com/backend

# 安装 Go（如果未安装）
# Ubuntu/Debian
sudo apt install golang-go -y

# 下载依赖
go mod download

# 编译
CGO_ENABLED=0 go build -ldflags="-s -w" -o webcheckly main.go
```

### 方式三：使用 Docker 打包（可选）

如果需要 Docker 部署，可以创建 `Dockerfile`：

```dockerfile
# 构建阶段
FROM golang:1.21-alpine AS builder

WORKDIR /build

# 复制 go mod 文件
COPY go.mod go.sum ./
RUN go mod download

# 复制源代码
COPY . .

# 编译
RUN CGO_ENABLED=0 GOOS=linux go build -ldflags="-s -w" -o webcheckly main.go

# 运行阶段
FROM alpine:latest

RUN apk --no-cache add ca-certificates tzdata

WORKDIR /app

# 从构建阶段复制二进制文件和迁移文件
COPY --from=builder /build/webcheckly .
COPY --from=builder /build/migrations ./migrations

EXPOSE 8080

CMD ["./webcheckly"]
```

构建 Docker 镜像：

```bash
cd backend
docker build -t webcheckly-backend:latest .
```

## 🚀 部署步骤

### 1. 上传文件

**从 Windows 上传到 Linux 服务器：**

**方法1: 使用 SCP (需要安装 OpenSSH 客户端)**

```powershell
# Windows PowerShell
# 如果使用压缩包
scp deploy\webcheckly-backend.zip user@server:/tmp/

# 如果直接上传目录（需要安装 rsync 或使用 WinSCP）
# 使用 WinSCP 图形界面工具上传整个 deploy\webcheckly-backend 目录
```

**方法2: 使用 WinSCP（推荐，图形界面）**

1. 下载并安装 [WinSCP](https://winscp.net/)
2. 连接到 Linux 服务器
3. 将 `deploy\webcheckly-backend` 目录上传到服务器的 `/tmp/` 目录
4. 或直接上传到 `/opt/webcheckly/`（需要 sudo 权限）

**方法3: 使用 SFTP 客户端**

使用 FileZilla、MobaXterm 等工具上传文件。

**在 Linux 服务器上解压和配置：**

```bash
# SSH 登录到服务器
ssh user@server

# 如果上传的是压缩包，先解压
cd /tmp
unzip webcheckly-backend.zip
# 或
tar -xzf webcheckly-backend.tar.gz

# 移动到目标目录
sudo mv webcheckly-backend/* /opt/webcheckly/

# 确保所有权和权限
sudo chown -R webcheckly:webcheckly /opt/webcheckly
sudo chmod +x /opt/webcheckly/webcheckly
sudo chmod +x /opt/webcheckly/start.sh  # 如果有启动脚本
```

**方法4: 使用 rsync（如果 Windows 上安装了 rsync）**

```powershell
# Windows PowerShell（需要安装 Git for Windows 或 WSL）
# 通过 WSL 使用 rsync
wsl rsync -avz deploy/webcheckly-backend/ user@server:/opt/webcheckly/
```

### 2. 配置环境变量

将应用设置为从配置目录读取 `.env` 文件，或者在 systemd 服务中加载环境变量。

### 3. 创建 systemd 服务（推荐）

创建服务文件：

```bash
sudo nano /etc/systemd/system/webcheckly.service
```

服务配置：

```ini
[Unit]
Description=WebCheckly Backend Service
After=network.target postgresql.service
Requires=postgresql.service

[Service]
Type=simple
User=webcheckly
Group=webcheckly
WorkingDirectory=/opt/webcheckly
Environment="PATH=/usr/local/bin:/usr/bin:/bin"

# 加载环境变量文件
EnvironmentFile=/opt/webcheckly/config/.env

# 执行命令
ExecStart=/opt/webcheckly/webcheckly

# 重启策略
Restart=always
RestartSec=10

# 资源限制
LimitNOFILE=65535
MemoryLimit=2G
CPUQuota=400%

# 日志
StandardOutput=journal
StandardError=journal
SyslogIdentifier=webcheckly

[Install]
WantedBy=multi-user.target
```

### 4. 启动服务

```bash
# 重新加载 systemd
sudo systemctl daemon-reload

# 启动服务
sudo systemctl start webcheckly

# 设置开机自启
sudo systemctl enable webcheckly

# 检查状态
sudo systemctl status webcheckly

# 查看日志
sudo journalctl -u webcheckly -f
```

### 5. 验证部署

```bash
# 检查服务是否运行
curl http://localhost:8080/health

# 应该返回：
# {"status":"ok","timestamp":1234567890}

# 检查端口监听
sudo netstat -tlnp | grep 8080
# 或
sudo ss -tlnp | grep 8080
```

## 🔍 故障排除

### 常见问题

#### 1. 服务无法启动

**检查日志：**
```bash
sudo journalctl -u webcheckly -n 50
```

**常见原因：**
- 环境变量未正确配置
- 数据库连接失败
- 端口被占用
- 文件权限问题

**解决方案：**
```bash
# 检查端口占用
sudo lsof -i :8080

# 检查文件权限
ls -la /opt/webcheckly/webcheckly

# 手动测试启动
sudo -u webcheckly /opt/webcheckly/webcheckly
```

#### 2. 数据库连接失败

```bash
# 测试数据库连接
psql -h localhost -U webcheckly_user -d webcheckly

# 检查 PostgreSQL 状态
sudo systemctl status postgresql

# 查看 PostgreSQL 日志
sudo tail -f /var/log/postgresql/postgresql-*.log
```

#### 3. 外部工具未找到

```bash
# 检查工具是否在 PATH 中
which httpx
which katana

# 测试工具
httpx -version
katana -version

# 如果不在 PATH 中，创建符号链接
sudo ln -s /path/to/httpx /usr/local/bin/httpx
```

#### 4. 内存不足

```bash
# 查看内存使用
free -h

# 调整 systemd 内存限制
sudo systemctl edit webcheckly

# 在打开的编辑器中添加：
[Service]
MemoryLimit=4G
```

#### 5. 迁移失败

```bash
# 检查迁移文件
ls -la /opt/webcheckly/migrations/

# 手动执行迁移（如果需要）
psql -h localhost -U webcheckly_user -d webcheckly < migrations/001_create_users_table.up.sql

# 查看迁移记录
psql -h localhost -U webcheckly_user -d webcheckly -c "SELECT * FROM schema_migrations;"
```

## 🔄 更新升级

### 1. 备份数据

```bash
# 备份数据库
pg_dump -h localhost -U webcheckly_user webcheckly > backup_$(date +%Y%m%d_%H%M%S).sql

# 备份配置文件
sudo cp /opt/webcheckly/config/.env /opt/webcheckly/config/.env.backup
```

### 2. 更新步骤

**在 Windows 开发环境：**

```powershell
# 1. 重新编译新版本
cd C:\Users\red\Desktop\WebCheckly.com\backend
$env:GOOS = "linux"
$env:GOARCH = "amd64"
$env:CGO_ENABLED = "0"
go build -ldflags="-s -w" -o webcheckly main.go

# 2. 上传到服务器
scp webcheckly user@server:/tmp/
```

**在 Linux 服务器上：**

```bash
# 1. 停止服务
sudo systemctl stop webcheckly

# 2. 备份旧版本
sudo cp /opt/webcheckly/webcheckly /opt/webcheckly/webcheckly.backup

# 3. 替换新版本
sudo mv /tmp/webcheckly /opt/webcheckly/
sudo chown webcheckly:webcheckly /opt/webcheckly/webcheckly
sudo chmod +x /opt/webcheckly/webcheckly

# 4. 更新迁移文件（如果有）
# 从 Windows 上传迁移文件
# scp -r migrations/* user@server:/tmp/migrations/
sudo cp -r /tmp/migrations/* /opt/webcheckly/migrations/
sudo chown -R webcheckly:webcheckly /opt/webcheckly/migrations

# 5. 启动服务
sudo systemctl start webcheckly

# 6. 验证
sleep 5
curl http://localhost:8080/health
sudo systemctl status webcheckly
```

### 3. 回滚（如果出现问题）

```bash
# 停止服务
sudo systemctl stop webcheckly

# 恢复旧版本
sudo mv /opt/webcheckly/webcheckly.backup /opt/webcheckly/webcheckly

# 恢复数据库（如果需要）
psql -h localhost -U webcheckly_user webcheckly < backup_*.sql

# 启动服务
sudo systemctl start webcheckly
```

```nginx
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    # 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com www.yourdomain.com;

    # SSL 配置
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # 日志
    access_log /var/log/nginx/webcheckly_access.log;
    error_log /var/log/nginx/webcheckly_error.log;

    # 客户端最大请求体大小
    client_max_body_size 10M;

    location / {
        proxy_pass http://webcheckly_backend;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # SSE 支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }

    # 健康检查端点
    location /health {
        proxy_pass http://webcheckly_backend/health;
        access_log off;
    }
}
```
