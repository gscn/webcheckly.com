
# 一、新需求功能模块补充

**核心原则**
* Lighthouse：负责「质量 & 体验」
* httpx：负责「可用性 & 存活性 & 扫描」


## 模块 1：性能体验检测（Lighthouse 主体）

> 用 Lighthouse，但**不原样展示**

---

### 1.1 核心 Web Vitals 面板

**指标**

* LCP
* CLS
* INP / TBT

**处理方式**

* 原始数据 → 中文解释
* 是否达标
* 风险等级

---

### 1.2 LCP / TBT 深度诊断

**功能**

* LCP 元素定位
* 阻塞资源列表
* JS 长任务分析

**实现**

* Lighthouse JSON
* 自己解析 audits + traces

---

### 1.3 性能回归对比（Diff）

**功能**

* 同一 URL 不同时间对比
* 指标变化趋势

**适合**

* 订阅用户
* CI / 发布前检测

---

## 模块 2：SEO 技术合规检测（非内容）

> 这是你最容易获客的模块

---

### 2.1 基础 SEO 技术项

**检测项**

* title / meta description
* viewport
* robots.txt
* canonical
* indexability

**来源**

* Lighthouse SEO audits
* httpx + HTML parser 补充

---

### 2.2 SPA / JS 渲染可见性检测（特色）

**功能**

* SSR / CSR DOM 对比
* 首屏是否有有效文本
* 链接是否可爬取

**技术**

* Lighthouse 渲染后 DOM
* httpx 原始 HTML

---

## 模块 3：前端安全

### 3.1 第三方脚本风险扫描

**功能**

* 脚本来源统计
* 阻塞脚本
* 风险提示（ads / unknown）

**来源**

* Lighthouse third-party-summary
* DOM script 分析

---

## 模块 4：可访问性（Accessibility）合规模块

> 可以作为 **B 端 / 海外付费功能**

---

### 4.1 表单无障碍检测

**检测**

* label 绑定
* aria-* 合法性
* tabindex

**来源**

* Lighthouse accessibility audits

---

### 4.2 对比度 & 可读性

**输出**

* 不合规文本列表
* 对应 WCAG 等级
